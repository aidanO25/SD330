User Design Principles

  • Know who your target users are
  • Implement user feedback into the new design
  • Ensure consistency with what your usuers expect
  • Understand users needs

Cognative Principles

  • Acknowledge people have different ideas of doing things
  • Think how a new user would interact with features when designing them
  • Put yourself in the mindset of the user when desiging different pages to navigate to
  • Make sure that you make pages accessable to visual impariments such as color blindness
  • Make features easily understandable by using metaphores such as a shopping cart for keeping items in while you continue to shop
  • Think about the way users will interact with your sight having certain cognative conditions such as ADHD
  • Respect the focus of other users, number of notifications given is an example of this

Cultural and Societal Principles

  • Ensure atequite knowledge on the culture which you are developing a product for
  • Know certain phrases that may be viewed differently, either posotively or negatively, in one culture than the one you are from
  • Understand the laws of the area that you plan to develop an application for
  • Tailor the application that sutes the intended culture

User Design Principles

  • Put the user first
  • Know who your target users are
  • Implement user feedback into the new design
  • Ensure consistency with what your usuers expect
  • Understand users needs
  • Reduce users' experiences of anxiety
  • Help users gain confidence and mastery and feel in control, this can be done by keeping things consistent

Cognative Principles

  • Acknowledge people have different ideas of doing things
  • Think how a new user would interact with features when designing them
  • Point the user in the right direction such as making specific functions stand out from the rest which minimizes confusion
  • Simplify features and pages as much as possible
  • Use metaphores to allow users to hopefully understand what that feature does
  • Create user stories and scenarios to understand how a user would interact with the system
  • Balance the amount of information given on each page to minimize page congestion and user fustration

Social Principles

  • Design your systems, more specifically AI chat bots, that makes the user feel they are talking to a real person (if that is the intent)
  • Confirm that your application, an example is FaceTime, can reliably transmit information so face to face videos and audio are not low quality
  • Make the user feel comfortable when interacting with your system/application
  • Ensure you have protection against innapropriate messages between users, especially when the intended audience are young children
  • Implement features that allow users to connect

Graphical Principles

  • Design your application to be visibly asethetic
  • Align information in a neat and ordly fassion that isn't overstimmulating to the user
  • Use colors that are similar to one another rather than obnoxious ones when it isn't needed
  • Ensure text is easily readable
  • Add information that doesn't properly fit on the page to a new page allowing the originalto be de-cluttered
  • Emphasize pictures or text that you want to make sure the user sees

Prototyping Principles

  • Create a simple moc up, other wise known as a low-fidelity prototype which saves you money and time
  • Implement teh features you plan to have for the final product into your prototype for users to test them
  • Gather user feedback, both posotive and negative, to further advance your prototype
  • List out all the ideas you have for the product by creating a conceptual model or story board
  • Develop a high-fidelity prototype when you get closer to teh final product
